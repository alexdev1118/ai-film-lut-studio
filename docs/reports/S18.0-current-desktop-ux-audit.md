# S18.0 当前桌面体验审计

审计日期：2026-07-16。实际 Chrome 尺寸覆盖 1366×768、1440×900 和 1920×945，手机端不作为本阶段门禁。

## 结论

首页在五秒内能够表达“本地生成创意 LUT”的用途，但旧路径要求新手过早理解 Gamma、Gamut、CST、Range、POST、CAMMON 与 Hash。真正高风险的不是参数不会调，而是用户不知道上传静帧当前是什么像素状态，以及创意 LUT 是否包含 Log 技术还原。

## 三条真实路径

1. 完全不知道素材状态：必须允许选择“不确定”，继续实验，同时明确结果不能直接视为可靠 Rec.709 POST LUT。
2. 知道拍摄来源为 Sony S-Log3：原始拍摄来源可以记录，但不能因此推断当前上传静帧仍是 S-Log3；若静帧已经从 DaVinci 导出为 Rec.709，两者必须分开显示。
3. 已从 DaVinci 导出正常 Rec.709 静帧：应直接进入风格、强度、预览和导出，不应要求先配置相机目录。

## 最迷惑与最容易出错

- 原始相机来源和当前静帧像素状态混在同一区域。
- POST 创意 LUT、技术转换和相机监看 LUT 同屏竞争注意力。
- Hash、MAE、Round-Trip 属于诊断，不属于首次任务。
- Log 直接套 POST、重复 CST、同时降低网站强度和 Key Output 是最高风险误用。
- 自动仿色是浏览器本地启发式色彩分析，不应暗示云端深度学习模型。

## 保留能力

本地素材、DPX、结构化风格、统一 RGB Core、Preview/Cube 一致性、Cube 校验、下载 Artifact、相机证据与 Round-Trip 都值得保留。正确做法是分层显示，不是删除专业能力。

## 已实施方向

- 默认进入快速模式，专业模式保留完整能力。
- 快速模式先询问素材看起来是 Log、正常颜色或不确定。
- Log 未还原路径阻止可靠 POST 导出声明。
- Hash 与 Round-Trip 隐藏到专业模式或验证详情。
- 风格库支持同图比较四个候选。
- 导出说明根据 DaVinci、Premiere、Final Cut、相机监看或其他 Cube 软件变化。
