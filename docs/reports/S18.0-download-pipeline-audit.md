# S18.0 POST LUT Download Pipeline Audit

Retrieved and audited: 2026-07-16

## Result

The POST LUT download core gate passes. Artifact construction, parsing, validation, hashing, retry stability, style differentiation, and the browser anchor/Blob code path are verified. Chrome automation did not capture the Blob download event and Windows file landing remains pending because the Windows browser URL policy prevents automated inspection of Chrome system pages. These two auxiliary items do not block product development.

## Call Chain

1. `Workspace.handleExportCube()` selects the current `PreviewResult.preparedPostLut`.
2. `createPreparedPostLutDownloadArtifact()` calls the single `buildCubeDownloadArtifact()` implementation.
3. `buildCubeDownloadArtifact()` validates and parses the exact `preparedPostLut.cubeResult.content`, verifies parameter and Cube hashes, and creates the Blob.
4. `requestPostLutDownload()` calls `triggerCubeDownloadArtifact()`.
5. `triggerCubeDownloadArtifact()` creates the object URL and hidden anchor, appends the anchor, calls `click()`, removes the anchor, then schedules URL revocation after 10 seconds.
6. Workspace, ExportResult, and History retry paths all reuse `WorkspaceProvider.lastCubeDownloadArtifact`; they do not regenerate another Cube.

## Audit Questions

1. **Export function:** Workspace calls `createPreparedPostLutDownloadArtifact()` and `requestPostLutDownload()`; compatibility service `downloadPreparedPostLut()` composes the same two functions.
2. **User gesture:** the old path awaited preview preparation, parsing, and hashing before `anchor.click()`. The new path prebuilds and caches the Artifact during preview generation, so a normal export or retry calls `requestPostLutDownload()` synchronously from the button event when the current hashes match.
3. **Async blocking risk:** removed from the normal cached path. The fallback still prepares an Artifact asynchronously, but the UI exposes a synchronous retry using the verified cached Artifact.
4. **Blob content:** exactly `preparedPostLut.cubeResult.content`; Blob text equality and byte length are tested.
5. **Download attribute:** set to `artifact.filename` and required to end in `.cube`.
6. **Document insertion:** the hidden anchor is appended to `document.body` before `click()`.
7. **Early revocation:** fixed. Revocation is scheduled for 10 seconds after the click.
8. **Immediate revoke:** not present in the POST path. Failure cleanup revokes only when the request throws before successful scheduling.
9. **Filename:** generated by the existing POST naming contract and validated as a `.cube` name.
10. **UI-only success:** prevented by the explicit Artifact and trigger tests. The requested status is set only after `triggerCubeDownloadArtifact()` returns.
11. **Stale prepared data:** Workspace compares Artifact `cubeHash` and `parameterHash` with the current prepared result before reuse.
12. **Multiple-download permission:** the application requests one file per click. Browser policy or permission behavior is external and remains separately observable.
13. **Multiple export entries:** POST download and retry entries share one Artifact trigger. CAMMON keeps its separate format-specific export path.
14. **ExportResult consistency:** ExportResult and History do not regenerate Cube text; they request the same Provider Artifact used by Workspace.
