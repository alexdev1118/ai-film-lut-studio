# S17.3 Bundle Audit

## Scope

The audit covers the Release Candidate route graph and production Vite output. It does not change the Workspace provider, LUT core, color contracts, camera evidence, or export behavior.

## Before

| Asset | Minified | Gzip |
| --- | ---: | ---: |
| Main JavaScript | 563.80 kB | 165.91 kB |
| Main CSS | 83.78 kB | 14.76 kB |

Vite reported one chunk above the 500 kB warning threshold because all eight route pages were imported synchronously by `src/App.tsx`.

## Change

All route pages now use `React.lazy()` behind one `Suspense` boundary. `WorkspaceProvider` remains above the route content, so navigating between pages keeps the existing session state. No dependency was added.

## After

| Asset | Minified | Gzip |
| --- | ---: | ---: |
| Main JavaScript | 244.51 kB | 77.38 kB |
| Workspace route | 220.68 kB | 57.84 kB |
| Style Library route | 80.47 kB | 15.42 kB |
| Export Result route | 10.99 kB | 4.08 kB |
| Main CSS | 85.13 kB | 14.97 kB |

The largest production chunk is 244.51 kB, and Vite no longer reports a chunk-size warning. The main JavaScript payload is 319.29 kB smaller before gzip and 88.53 kB smaller after gzip. Browser regression checks confirmed route navigation, style selection, preview generation, export, history, and session state after the split. The Style Library chunk includes its 120-preview manifest metadata; PNG assets remain separate static files and are loaded only by the style route.

## Boundaries

- Test reports and local DaVinci renders remain outside the production import graph.
- Cube files are generated on demand and are not embedded as precomputed strings.
- Camera evidence stays in its route consumers and is not fetched from a remote service.
- Further vendor chunk tuning is deferred because the current split removes the warning without introducing configuration complexity.
