- Magic: `SDPX`; Big Endian; DPX version `V1.0`.
- Resolution: `3840 x 2160`; orientation `0`; one image element.
- Data offset: `8192`; declared size matches the actual file size.
- Descriptor `50` RGB; Data Sign `0`; bit depth `16`; Packing `0`; Encoding `0`.
- Transfer characteristic `1`; colorimetric specification `2`; these raw header values are retained as clues only and are not automatically interpreted as an official Rec.709, Gamma, or range contract.
- Line padding `0`; image padding `0`; no channel-order anomaly was observed.
- Decoder output: `2048 x 1152` local 8-bit preview, with opaque alpha for every decoded preview pixel.
- DPX A preview RGB min/max: `17/17/17` to `212/204/194`.

The existing DPX real-file regression assertion was corrected from an unsupported hard-coded `10-bit / Packing 1` expectation to the values proven by both real source files: `16-bit / Packing 0`. The DPX parsing and decoding implementation was not changed.
