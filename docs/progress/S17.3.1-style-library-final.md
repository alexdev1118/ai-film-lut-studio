# S17.3.1 Style Library Hotfix

Status: passed locally; no Git commit or push was performed.

## Reproduced Issue

The visible image area of a style card was not an actionable control. Clicking it did not open details or apply a style, which made the style library appear inert. The browser audit found all 12 definitions, all 48 strength previews, and all 72 scene previews on disk and available from `public/style-previews/`.

## Fix

- The preview image is now a keyboard-accessible button that opens the corresponding style detail.
- The style title is also an explicit detail trigger.
- Style application uses one `WorkspaceProvider` parameter state through `applyLutStyleToWorkspace`.
- The workspace now shows the selected style, parameter hash prefix, and final Cube hash prefix after application.
- An unavailable preview renders an explicit fallback rather than an empty card.

## Browser Evidence

- Desktop: 12 style cards rendered; the Print 2383 preview button opened its detail view.
- Style detail to workspace: `自然肤色` navigated to `/workspace?style=natural-skin&strength=35` and completed its provider update after the lazy workspace loaded.
- 390 by 844: 12 cards rendered, the workspace showed application proof, and no horizontal overflow was detected.
- Project console: no local application warnings or errors in the final browser QA tab.
