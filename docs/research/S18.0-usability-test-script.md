# S18.0 桌面可用性测试脚本

## 设备与环境

- Windows 或 macOS 桌面浏览器，最小目标视口 1366 × 768。
- 使用测试组织拥有明确授权的静帧和参考图。
- 不预装浏览器扩展，不打开开发者工具。

## 主任务

向参与者展示以下情境，之后不补充术语解释：

> 我用 Sony 相机拍了一段灰色的 S-Log3 视频。我喜欢一张参考图的颜色，准备在 DaVinci Resolve 使用，但不知道 CST 是什么。请用页面生成一个适合后期使用的 LUT，并找到正确的使用步骤。

## 观察点

1. 是否在 5 秒内理解产品用途。
2. 是否选择“灰、低对比的 Log”。
3. 是否知道未技术还原的 Log 不能被可靠地直接制作成 POST 创意 LUT。
4. 是否找到相机来源和当前静帧像素状态的区别。
5. 是否选择参考图路径并上传合法静帧。
6. 是否能在预览前选择自然、标准、浓郁或完整强度。
7. 是否理解 DaVinci 中先做技术还原再叠加创意 LUT。
8. 是否完成下载并找到重新下载与文件验证信息。
9. 是否错误地同时降低网页强度和 DaVinci Key Output。
10. 从开始到下载使用了多少主要步骤。

## 追加任务

- 切换到专业模式，找到 Gamma、Gamut、Range 和 Cube Hash。
- 返回快速模式，确认已选素材、参数和风格没有丢失。
- 打开风格库，对比四个风格并选择其中一个带入工作台。
- 将目标软件切换为 Premiere Pro，确认导出指南发生变化。

## 结果等级

- 通过：独立完成且没有产生错误色彩链声明。
- 有帮助通过：需要一次不包含答案的导航提示。
- 未通过：需要解释 CST、节点顺序或直接 Log 风险后才能继续。
- 技术阻塞：页面崩溃、媒体无法读取、预览与导出状态不一致或下载链路没有执行。
