# S18.0 用户访谈提纲

## 开场

说明本次研究评估产品，不评估参与者。主持人不提前解释 Gamma、Gamut、CST 或 LUT 节点顺序。

## 背景问题

1. 你通常拍照片、视频，还是两者都拍？
2. 你使用哪些相机、手机和剪辑软件？
3. 你是否拍过 Log？当时如何把画面恢复到正常颜色？
4. 你最近一次使用 LUT 的目标是什么？
5. 你从哪里获得 LUT，又如何判断它适合当前素材？

## 任务后追问

1. 第一次看到页面时，你认为它能帮你完成什么？
2. 哪一步最让你不确定？
3. “灰、低对比的 Log”“正常颜色”“不确定”是否覆盖你的素材状态？
4. 你如何理解“技术还原”和“创意 LUT”的区别？
5. 你是否知道导出的 LUT 应在 DaVinci 或 Premiere 的什么位置使用？
6. 你是否信任网页预览？什么证据会提高或降低信任？
7. 风格并排比较是否比逐个打开详情更高效？
8. 你更希望产品自动决定，还是保留手动参数？为什么？
9. 本地处理和不上传媒体是否重要？
10. 哪项能力值得你持续使用或付费？

## 专业用户追问

1. 输入输出契约是否足够明确？
2. Hash、Round-Trip 和技术转换证据应该默认显示还是折叠？
3. 你会如何验证预览、Cube 和 NLE Render 的一致性？
4. 哪些相机资料必须有官方来源才能被信任？
5. 插件的最低价值是传递静帧、写入 LUT，还是完整节点自动化？

## 禁止引导

- 不问“你是不是觉得快速模式更简单”。
- 不把营销承诺当作用户答案。
- 不向参与者暗示正确的节点顺序。
- 不把计划招募人数写成已完成样本。
